Document Structure

The structure of an HTML document is crucial for creating a well-organized and accessible website. It includes the use of proper elements such as the doctype, head, and body to define the structure and hierarchy of the content.

Document Structure Best Practices

  • Use a doctype to specify the version of HTML being used
  • Use headings (h1-h6) to create a hierarchical structure for the content
  • Use proper semantic elements such as header, main, article, and footer to define the sections of the page
  • Use a lang attribute to specify the language of the page
  • Use alt attributes for images to improve accessibility

Document Structure Example

          
  <!DOCTYPE html>
  <html lang="en">
  <head>
    <meta charset="UTF-8">
    <title>My Website</title>
  </head>
  <body>
    <header>
      <nav>
        <ul>
          <li><a href="#home">Home</a></li>
          <li><a href="#about">About</a></li>
          <li><a href="#services">Services</a></li>
          <li><a href="#contact">Contact</a></li>
        </ul>
      </nav>
    </header>
    <main>
      <article>
        <h1>Welcome to My Website!</h1>
        <img src="image.jpg" alt="A picture of something">
        <p>This is the home page of my website. Here you can learn more about me and the services I offer.</p>
      </article>
    </main>
    <footer>
      <p>Copyright Â© 2021 My Website</p>
    </footer>
  </body>
  </html>

In this example, we have used a doctype to specify that we are using HTML5, a lang attribute to specify that the language of the page is English, and proper semantic elements such as header, main, article, and footer to define the sections of the page. We have also used headings (h1-h6) to create a hierarchical structure for the content and an alt attribute for the image to improve accessibility. Additionally, by using a navigation menu within the header element, we can easily provide links to the different sections of the website and aid in user navigation. Proper document structure is essential for creating a well-organized and accessible website and should be considered when creating any HTML document.
