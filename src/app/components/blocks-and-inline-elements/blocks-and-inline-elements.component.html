Blocks and Inline Elements

HTML elements can be classified as either block-level or inline elements. Block-level elements take up the full width of their parent container and create a new block formatting context, while inline elements only take up as much width as necessary and do not create a new block formatting context.

Block-level Elements

Block-level elements include elements such as <div>, <p>, <h1-h6>, <header>, <footer>, <section>, and <article>. These elements take up the full width of their parent container and create a new block formatting context, meaning that they start on a new line and take up the full width of their parent container.

Inline Elements

Inline elements include elements such as <span>, <a>, <img>, and <strong>. These elements only take up as much width as necessary and do not create a new block formatting context, meaning that they will be on the same line as any other inline elements around them.

Blocks and Inline Example

          
  <header>
    <h1>Welcome to My Website!</h1>
    <nav>
      <a href="#">Home</a>
      <a href="#">About</a>
      <a href="#">Contact</a>
    </nav>
  </header>
  <section>
    <p>This is a <strong>block-level</strong> element. It takes up the full width of its parent container and creates a new block formatting context.</p>
    <p>This is an <strong>inline</strong> element. It only takes up as much width as necessary and does not create a new block formatting context.</p>
  </section>

In this example, we have used block-level elements such as <header>, <h1>, and <section> to create the structure and layout of the page. We also used inline elements such as <nav> and <a> to create the navigation menu within the header. Additionally, within the section, we have used <p> as a block-level element and <strong> as an inline element. The <p> element takes up the full width of its parent container and creates a new block formatting context, while the <strong> element only takes up as much width as necessary and does not create a new block formatting context. By understanding the difference between block-level and inline elements, we can create a well-structured and organized webpage.
